Validation Check - Lab 2

FearlessRecords.com

Fearless Records is a record company and their site consists of news updates, band information, and videos.

Number of errors:

  • 20 errors in the validator.nu
  • HTML Lint Validator couldn’t continue because of an error in the HTML

A lot of errors occurred because entities weren’t used for the ampersand. This can be easily fixed by replacing the ampersands with it's entity. The other errors occurred because of missing alt attribute values. All the images have alt=” “, but aren’t given any values. The HTML Lint Validator couldn’t be continued because inline elements, <strong> <p> <br />, were used in the values for title attributes.

PureNoise.net

Pure Noise Records is also a record company and their site consists of news updates, band information, and videos.

Number of errors:

  • 25 errors in the validator.nu
  • HTML Lint Validator couldn’t continue because of an error in the HTML

End tags aren’t in order throughout the page. (ex. <p><font color="#666666">No More Than This</p></font> ) So the end tags just need to be flipped around to fix that. This is also the reason why the HTML Lint Validator couldn’t continue the validation. None of the images have alt attributes either, so those need to be added to each image. Also, for some elements, spaces are missing in between separate attributes.

EyeSuckInk.com

Eye Suck Ink is a portfolio website for artist Alex Pardee. It’s basically just an image gallery of all his work.

Number of errors:

  • 1 error in the validator.nu
  • 0 errors in the HTML Lint Validator

One of the images is missing the alt attribute, so screen readers won’t be able to read it. An alt attribute just needs to be added to the img element.

NineLionDesign.com

Nine Lion Design is a portfolio website for Chikezie Ejiasi, an interaction designer.

Number of errors:

  • 5 errors in validator.nu
  • 7 errors in the HTML Lint Validator

A majority of the errors are a result of the doctype. It has a HTML5 doctype specified and a transitional HTML 4.01 doctype. Errors that are shown would be okay in HTML5, but because he also specified HTML 4.01, the validator shows them as errors. The meta tags aren’t closed which is okay in HTML 5, but not for HTML 4.01. There are also HTML 4.01 errors for the closing of image elements.
